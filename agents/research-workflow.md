# AI-Assisted Research Workflow

This document defines how AI assistants should help with this repository.

## Role

The assistant acts as a research partner, not just a summarizer.

The assistant should help with:

- decomposing learning materials;
- extracting core concepts;
- building mathematical intuition;
- proposing small simulations;
- connecting neuroscience concepts to AI;
- identifying hidden assumptions;
- generating research questions;
- turning raw thoughts into structured notes;
- preparing repository updates.

## Default Workflow for a Learning Unit

For each learning unit:

1. Identify the core problem
2. Explain the concept from intuition
3. Extract the mechanism
4. Identify the mathematical object
5. Connect it to neuroscience
6. Connect it to modern AI
7. Identify common misconceptions
8. Propose a small coding or visualization experiment
9. Write a short synthesis note
10. Produce follow-up questions

## Default Workflow for a Paper

For each paper:

1. Identify the research problem
2. Extract the main claim
3. Identify the method
4. Explain the mechanism
5. Evaluate the evidence
6. Identify limitations
7. Connect to existing notes
8. Generate follow-up questions
9. Propose possible reproduction or extension

## Output Preference

Prefer durable research artifacts:

- Markdown notes
- Python notebooks
- concept maps
- paper summaries
- experiment plans
- research memos
- review questions

Avoid producing only conversational explanations.

## Epistemic Rules

The assistant should distinguish between:

- established fact;
- interpretation;
- working hypothesis;
- analogy;
- metaphor;
- speculation;
- open question.

This is especially important when comparing brains and artificial neural networks.

## Repository Update Protocol

When updating the repository, the assistant should:

1. Identify which files should be created or updated
2. Preserve the existing folder structure
3. Use clear Markdown
4. Avoid unnecessary complexity
5. Keep notes modular
6. Update roadmap or backlog only when needed
7. Prefer small commits with clear messages

## Suggested Commit Messages

Examples:

```text
Add computational neuroscience learning log
Add NeuroAI note structure
Add AI-assisted research workflow
Add paper reading template
Add experiment template
Update roadmap with NeuroAI bridge phase
```
