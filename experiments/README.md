# Experiments

This folder contains small simulations, notebooks, prototypes, and mini-benchmarks.

## Purpose

Experiments are used to convert abstract concepts into executable understanding.

The goal is not to build large projects immediately. The goal is to create small, runnable demonstrations that clarify mechanisms.

## Experiment Types

### 1. Concept Simulations

Small simulations for understanding computational neuroscience concepts.

Examples:

- leaky integrate-and-fire neuron;
- spike train generation;
- population coding;
- attractor dynamics;
- recurrent neural network dynamics;
- simple reinforcement learning agents.

### 2. Neural Data Analysis

Small notebooks for analyzing synthetic or real neural data.

Examples:

- tuning curve fitting;
- PCA on neural population activity;
- decoding stimulus variables;
- encoding model regression;
- dimensionality reduction.

### 3. NeuroAI Probes

Small experiments connecting neural concepts to AI models.

Examples:

- representation similarity analysis;
- probing hidden states;
- comparing model features with behavioral variables;
- toy brain-model alignment experiments.

## Experiment Template

Each experiment should include:

1. Core question
2. Minimal setup
3. Variables
4. Mechanism
5. Expected result
6. Code
7. Interpretation
8. Failure modes
9. Follow-up questions

## Rule

Prefer small experiments that can be understood deeply over large experiments that are hard to interpret.
