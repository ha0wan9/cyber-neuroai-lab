# Contributing to Cyber NeuroAI Lab

This repository is primarily a personal learning and research lab.

The contribution protocol is designed to preserve clarity, traceability, and intellectual honesty.

## Commit Types

Use clear commit messages that describe the knowledge artifact being added or changed.

Examples:

```text
Initialize Cyber NeuroAI Lab structure
Add first computational neuroscience learning log
Add concept note on neural encoding
Add paper note template
Update NeuroAI roadmap
Add experiment plan for dynamical systems simulation
```

## Content Standards

When adding notes, prefer:

- clear claims;
- explicit assumptions;
- mechanism-level explanations;
- links to related concepts;
- open questions;
- failure modes;
- possible experiments.

Avoid:

- copying large passages from source materials;
- adding unverified claims as facts;
- turning metaphors into conclusions;
- creating too many folders before there is enough content;
- adding vague notes without a purpose.

## AI-Generated Content

AI-generated content should be treated as a draft.

Before committing important AI-generated notes:

- check the main claims;
- remove unsupported assertions;
- add references when possible;
- mark uncertain points as open questions;
- keep the author's own interpretation explicit.

## Note Lifecycle

The preferred note lifecycle is:

```text
Raw idea
→ learning log
→ structured note
→ concept note
→ experiment or research question
→ weekly review
→ public output or archived reference
```

## File Naming

Use lowercase kebab-case for files:

```text
encoding-model.md
neural-representation.md
dynamical-systems.md
2026-04-27-computational-neuroscience-start.md
```

## Review Questions

Before committing a learning artifact, ask:

1. What problem does this note solve?
2. What concept does it stabilize?
3. What assumptions does it contain?
4. What is still unclear?
5. Can it become an experiment, question, or public output?
