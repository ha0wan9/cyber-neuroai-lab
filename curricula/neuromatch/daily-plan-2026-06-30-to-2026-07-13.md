# Pre-NeuroAI Daily Learning Plan (2-Week Compressed)

Date range: 2026-06-30 to 2026-07-13

Format: 2 weeks, 5 learning days per week, 4 hours per day (40 hours total).
Weekends (2026-07-04/05 and 2026-07-11/12) are rest days.

Purpose: recompress the pre-NeuroAI prep phase from the 15-day / 3-hour cycle
plan into a denser 10-day / 4-hour schedule that still produces one durable
repository artifact per day. This is a reorganization of
[`daily-plan-2026-06-18-to-2026-07-10.md`](daily-plan-2026-06-18-to-2026-07-10.md);
the original 5-cycle plan is preserved unchanged for reference.

Learner profile: AI/CV engineer transitioning toward NeuroAI and computational
cognitive neuroscience.

Style constraint: structure-first. Every day ends with a small written artifact,
map, stub, or review note that can be reused later.

Primary sources:

- Neuromatch Computational Neuroscience Coursebook: <https://compneuro.neuromatch.io/>
- Neuromatch Computational Neuroscience Schedule: <https://compneuro.neuromatch.io/tutorials/Schedule/daily_schedules.html>
- Neuromatch Deep Learning Coursebook: <https://deeplearning.neuromatch.io/tutorials/intro.html>
- Neuromatch NeuroAI Prerequisites: <https://neuroai.neuromatch.io/neuroai>

Per-tutorial source URLs are listed in
[`pre-neuroai-source-extract.md`](pre-neuroai-source-extract.md) and
[`source-registry.md`](source-registry.md).

## How the 15-day plan maps onto 10 days

| Old cycle (3h × 3 days = 9h) | New days (4h each) | Net change |
|---|---|---|
| Cycle 1: Measurements + Model Types | Day 1 | 3 days → 1 day (measurement + model-types merged) |
| Cycle 2: Model Fitting + GLM + DL basics | Days 2–3 | 3 days → 2 days |
| Cycle 3: Dimensionality Reduction + Representation | Days 4–5 | 3 days → 2 days |
| Cycle 4: Linear Systems + Neuron Models + Dynamics | Days 6–8 | 3 days → 3 days |
| Cycle 5: Decisions + Hidden Dynamics + RL + Causality | Days 9–10 | 3 days → 2 days |

The compression mostly absorbs the per-cycle "consolidation/review day" into the
working days; reviews now happen at the end of each week instead of each cycle.

## Week 1: Foundations

### Day 1: 2026-06-30 (Tue) — Measurements and Model Types

Source material:

- W0D0 Neuro Video Series.
- W1D1 Model Types.
- NeuroAI Prerequisites, only as a readiness lens.

Core question:

- What can neuroscience measure, and what does it mean for a model to explain
  that data?

4-hour structure:

- 0:00–0:30: Skim W0D0 video sequence; list biological scales (molecules, cells,
  circuits, systems, behavior).
- 0:30–1:20: Build a measurement table across spikes, calcium imaging, fMRI,
  EEG/MEG, behavior, and artificial model activations — what is directly
  measured, what is inferred, and at which spatial/temporal scale.
- 1:20–1:35: Break.
- 1:35–2:25: Extract W1D1 model categories (descriptive, mechanistic, normative,
  encoding, decoding, simulation); build a What/How/Why comparison table.
- 2:25–3:10: Add AI/CV analogies (classifier, feature extractor, generative
  model, simulator, optimizer, policy); note the difference between predicting
  and explaining.
- 3:10–4:00: Convert both tables into first-pass maps and mark open questions.

Required outputs:

- `notes/computational-neuroscience/measurement-modality-map.md`
- `notes/computational-neuroscience/model-types-what-how-why.md`

Review questions:

- What is the difference between neural activity, measurement, and interpretation?
- Which modality is closest to single-neuron computation?
- What is the difference between a model that predicts and one that explains?
- What CV assumption becomes dangerous when applied to neural data?

### Day 2: 2026-07-01 (Wed) — Model Fitting

Source material:

- W1D2 Model Fitting.
- DL bridge: PyTorch basics, gradient descent and Autograd, learning
  hyperparameters, optimization techniques.

Core question:

- What is being optimized when a model is fit to neural or behavioral data?

4-hour structure:

- 0:00–0:40: Summarize parameters, predictions, loss, likelihood (MSE vs MLE),
  bias-variance, and cross-validation.
- 0:40–1:30: Connect fitting to PyTorch training loops (forward, loss, backward,
  update); map gradient descent and hyperparameters onto the same picture.
- 1:30–1:45: Break.
- 1:45–2:40: Draw the model-fitting pipeline: data → objective → fitted
  parameters → evaluation.
- 2:40–3:25: Add failure modes (leakage, under/overfitting, misspecified
  objective, weak baseline) and how regularization changes interpretation.
- 3:25–4:00: Clean the map and mark open questions.

Required output:

- `notes/computational-neuroscience/model-fitting-map.md`

Review questions:

- What is the difference between fitting parameters and choosing a model class?
- Why is a loss function not automatically the scientific question?
- What is the role of a baseline model?
- Which parts of deep-learning training are just model fitting in another form?

### Day 3: 2026-07-02 (Thu) — GLMs, Encoding and Decoding

Source material:

- W1D3 Generalized Linear Models.
- DL bridge: biological vs. artificial neurons, deep MLPs, regularization
  (parts 1–2).

Core question:

- How do GLMs formalize encoding and decoding, and how do they ladder up to MLPs?

4-hour structure:

- 0:00–0:40: Define GLM components: input features, linear predictor, link
  function, response distribution, likelihood.
- 0:40–1:30: Compare a GLM with a one-layer network and an MLP; build the
  "from GLM to MLP" ladder (linear → nonlinear link → hidden layer → depth →
  regularization).
- 1:30–1:45: Break.
- 1:45–2:40: Write the encoding/decoding distinction using stimulus, neural
  response, latent state, and behavior; add CV examples
  (image→activation encoding, activation→category decoding).
- 2:40–3:30: Note what changes as a model becomes less interpretable but more
  expressive, and when expressivity becomes a scientific liability.
- 3:30–4:00: Clean links and mark open questions.

Required output:

- `notes/computational-neuroscience/encoding-decoding-note.md`

Review questions:

- What makes a model an encoding vs a decoding model?
- Why can accurate decoding fail to explain the brain's computation?
- How does a GLM differ from a generic network in assumptions and interpretability?
- When does expressivity become a scientific liability?

### Day 4: 2026-07-03 (Fri) — Dimensionality Reduction and Representation Geometry

Source material:

- W1D4 Dimensionality Reduction.
- DL bridge: Introduction to CNNs; cost functions and representation tasks.

Core question:

- Why does high-dimensional neural activity need low-dimensional structure?

4-hour structure:

- 0:00–0:40: Define dimensionality, population activity, variance, manifolds,
  latent variables, projections; what a principal component does and does not
  represent.
- 0:40–1:30: Compare PCA-style reduction with learned (CNN) representations;
  introduce feature maps, receptive fields, tuning, population code.
- 1:30–1:45: Break.
- 1:45–2:40: Draw a representation geometry map: data matrix, axes, distances,
  clusters, trajectories, latent dimensions.
- 2:40–3:30: Add a caution section on what dimensionality reduction can and
  cannot prove (information loss, misleading 2-D visual intuition).
- 3:30–4:00: Clean the map; list candidate comparisons (CNN vs visual cortex).

Required output:

- `notes/computational-neuroscience/representation-geometry-map.md`

Review questions:

- What is a neural population vector?
- What does a principal component represent, and what does it not?
- What information can be lost during dimensionality reduction?
- What controls would a CNN-vs-cortex representation comparison need?

### Day 5: 2026-07-06 (Mon) — Representation Comparison + Week 1 Review

Source material:

- DL bridge: modern ConvNets and transfer learning; facial-recognition
  embeddings; un/self-supervised representation learning.
- W1D4 Dimensionality Reduction as the neuroscience anchor.

Core question:

- How should a CV engineer compare artificial and biological representations
  without overclaiming?

4-hour structure:

- 0:00–0:40: List CV representation concepts: embeddings, invariance,
  equivariance, layer hierarchy, pretext tasks; map them to neuroscience terms.
- 0:40–1:30: Draft a misconception list about representation similarity,
  dimensionality, and interpretability; separate established fact,
  interpretation, analogy, and open question.
- 1:30–1:45: Break.
- 1:45–2:35: Add a mini comparison workflow: collect activations, align
  units/samples, reduce dimensions, compare geometry, test robustness.
- 2:35–3:20: Write open questions that could become paper-reading prompts and
  mark which methods need future runnable implementation.
- 3:20–4:00: Write the Week 1 review (stable concepts, fragile concepts,
  dependencies for Week 2).

Required outputs:

- `notes/neuroai/representation-learning-misconceptions.md`
- `learning-log/2026-07-06-week-1-review.md`

Review questions:

- Does similar geometry imply similar mechanism?
- What is the difference between a useful embedding and a biological explanation?
- Why is self-supervised learning relevant to NeuroAI, and where is the analogy weak?
- Which part of this week should become a runnable experiment later?

## Week 2: Dynamics, Inference, Learning, and Synthesis

### Day 6: 2026-07-07 (Tue) — Linear Systems and Sequence Models

Source material:

- W2D3 Linear Systems.
- DL bridge: introduction to processing time series.

Core question:

- How do linear systems explain temporal processing and state evolution?

4-hour structure:

- 0:00–0:40: Define state, input, output, impulse response, stability,
  eigenvalues, and linear approximation; cover Markov processes and
  autoregressive models.
- 0:40–1:30: Connect linear systems to sequence models: recurrence, state
  update, memory, temporal filtering.
- 1:30–1:45: Break.
- 1:45–2:40: Draft the linear-systems half of the dynamics concept map.
- 2:40–3:30: Add examples of what linear systems can approximate in neural and
  artificial systems, and what the linear approximation hides.
- 3:30–4:00: Clean and mark open questions.

Required output:

- `notes/computational-neuroscience/dynamics-concept-map.md` (linear-systems section)

Review questions:

- What is a system state, and why do eigenvalues matter for dynamics?
- How is recurrence different from feedforward computation?
- What does a linear approximation buy you, and what does it hide?

### Day 7: 2026-07-08 (Wed) — Biological Neuron Models

Source material:

- W2D4 Biological Neuron Models (LIF, input correlation, synaptic transmission;
  bonus STDP).

Core question:

- What does a biological neuron model explain that an artificial unit ignores?

4-hour structure:

- 0:00–0:40: Define membrane potential, threshold, leak, input current,
  refractory period, spike generation.
- 0:40–1:30: Compare artificial neuron, rate model, integrate-and-fire, and
  conductance-based models; note what is lost when spikes become firing rates.
- 1:30–1:45: Break.
- 1:45–2:45: Draft the LIF portion of the experiment stub: objective, minimal
  equations, expected plots, and a verification-command placeholder.
- 2:45–3:30: Add biological-realism tradeoffs to the dynamics concept map.
- 3:30–4:00: Clean the stub; note which question each neuron model suits.

Required output:

- `experiments/dynamics/lif-wilson-cowan-stub.md` (LIF section)

Review questions:

- What is being integrated in an integrate-and-fire neuron?
- What is lost when spikes are replaced by firing rates?
- Which neuron model is useful for which scientific question?

### Day 8: 2026-07-09 (Thu) — Dynamical Systems and Transformers

Source material:

- W2D5 Dynamical Systems (Neural Rate Models, Wilson-Cowan; bonus extension).
- DL bridge: attention and transformers (and pretraining/fine-tuning bonus).

Core question:

- How do population dynamics connect neural systems, RNNs, and modern sequence
  models?

4-hour structure:

- 0:00–0:40: Define fixed point, attractor, phase plane, trajectory, stability,
  oscillation, bifurcation.
- 0:40–1:30: Compare dynamical-systems views of RNNs with transformer sequence
  processing at a high level; mark where the transformer analogy is weakest.
- 1:30–1:45: Break.
- 1:45–2:45: Add the Wilson-Cowan portion to the experiment stub: objective,
  variables, parameters, expected phase-plane behavior, verification placeholder.
- 2:45–3:30: Finish the dynamics concept map (fixed points, attractors,
  oscillations).
- 3:30–4:00: Clean both artifacts.

Required outputs:

- `notes/computational-neuroscience/dynamics-concept-map.md` (complete)
- `experiments/dynamics/lif-wilson-cowan-stub.md` (Wilson-Cowan section)

Review questions:

- What is the difference between a trajectory and a representation?
- How does an attractor help explain memory or decision states?
- What is the weakest part of the transformer analogy to biological dynamics?

### Day 9: 2026-07-10 (Fri) — Decisions, Hidden Dynamics, and RL

Source material:

- W3D2 Bayesian Decisions.
- W3D3 Hidden Dynamics.
- W3D4 Reinforcement Learning.
- DL bridge: basic reinforcement learning; RL for games / Deep RL thinking.

Core question:

- How do uncertainty, hidden state, and reward shape behavior — and what carries
  over to deep RL?

4-hour structure:

- 0:00–0:40: Define prior, likelihood, posterior, decision variable, uncertainty;
  hidden state, latent dynamics, observation model, inference.
- 0:40–1:30: Start the NeuroAI bridge map with a decision-making branch and a
  hidden-dynamics branch; add AI/CV transfer notes (probabilistic inference,
  state estimation, uncertainty calibration).
- 1:30–1:45: Break.
- 1:45–2:40: Define state, action, reward, value, policy, prediction error,
  exploration, credit assignment; compare neuroscience RL with deep RL.
- 2:40–3:30: Add an RL / Deep RL branch to the bridge map.
- 3:30–4:00: List failure modes when importing deep RL concepts into
  neuroscience.

Required output:

- `notes/neuroai/neuroai-bridge-map.md` (decisions, hidden-dynamics, RL branches)

Review questions:

- What is the difference between uncertainty in the world and in the model?
- Why can behavior reveal latent variables?
- What is a reward prediction error, and why is credit assignment central?
- Where does the animal-learning / deep-RL analogy break down?

Source material:

- W3D5 Network Causality.
- NeuroAI Prerequisites.
- All prior Week 1 and Week 2 artifacts.

Core question:

- What foundation is now strong enough to begin NeuroAI work, and what remains a

4-hour structure:

- 0:40–1:30: Connect all branches into a single bridge: measurement → modeling →
